视觉组开发代码需要用一个版本控制工具，因此需要懂一些Git和Github作为团队协作的操作。这篇博客主要基于我们战队今年开源项目为实例演示团队合作流程。

首先大家需要掌握基础的Git操作。

<!-- more -->

由于暂时代码开发不需要用到多分支，所以我们的主仓库只用了两个分支，一个main分支用来存储稳定版本代码，另一个是开发分支dev，用于开发代码。

团队协作流程大概是这样的：

1. 战队账号创建一个主仓库
2. 新建分支dev
3. 拉小伙伴进入项目

个人流程是这样的：

1. 个人账号同意战队账号邀请
2. 个人账号fork战队的项目
3. clone到本地开发
4. 要提交时先合并一下战队仓库最新的代码
5. 然后从本地提交给个人远程仓库
6. 网页上点击PR按钮，待管理员审核后，个人仓库代码合并入主仓库

## 队员流程

### 1. fork主仓库到自己仓库

### 2. 将个人仓库clone到本地

![获取clone地址](https://raw.githubusercontent.com/cyicz123/Picture/main/img/20201025134239.png)

```
git clone url
```

### 3. 设置远程地址简称

```
git remote add origin url # 这里的url是个人仓库的地址
git remote add upstream url # 这里的url是战队仓库的地址
```

### 4. fetch、merge和push

这三个命令是团队协作的关键。

首先解释一下，第三步里为什么要设置两个远程地址的简称。个人fork主仓库里的内容是将主仓库当时的内容拷贝一份到自己的个人仓库里。但是有个问题是，**当自己的个人代码开发完毕上传到个人远程仓库再PR给主仓库时，主仓库可能在个人拷贝后更新了版本。**这样个人上传的代码是基于旧版本开发的，那么就会有问题。

所以在个人将自己本地代码提交给远程仓库时，要先**手动更新一下个人远程仓库保持和战队主仓库同步。**然后PR给主仓库。

具体步骤如下：

#### 1. 开发代码前更新一下个人仓库

![fetch](https://raw.githubusercontent.com/cyicz123/Picture/main/img/20201025135600.png)

`upstream`代表战队仓库地址，`dev`代表要拉取的相应分支（这里拉取的是`deb`分支）

![merge](https://raw.githubusercontent.com/cyicz123/Picture/main/img/20201025135957.png)

将刚刚拉取的远程内容合并到本地当前分支，`FETCH_HEAD`是git里代表刚刚用fetch拉取的内容。

#### 2. 开发代码

开发完后，使用`git add .` `git commit -m "提交备注"`来提交版本。

#### 3. 重复第一步

再次拉取战队仓库最新代码，因为个人开发过程中战队仓库可能有修改。

#### 4. 提交本地代码至个人远程仓库

![push](https://raw.githubusercontent.com/cyicz123/Picture/main/img/20201025140622.png)

`origin`代表个人远程仓库地址，第一个`main`代表本地分支名字，第二个`main`代表要上传到的远程分支名。

~~~
git push <远程主机名> <本地分支名>:<远程分支名>
~~~

#### 5. 在GitHub网页上点击PR

![Pull request](https://raw.githubusercontent.com/cyicz123/Picture/main/img/20201025141150.png)
